# 剪辑执行时的判断（第 6 步落地时带着这些）

> 主线在 SKILL.md（逐字稿是中心）。这份是主线走到「对着逐字稿把内容剪出来」时，落地要带的判断。
> 老 skill 的坑是**硬 apply**——机械套算法就翻车。下面每条是判断，不是可省的仪式。

## 六条判断

1. **剪辑服务内容，不套死算法。** 逐字稿已经是创作者挑过的内容，再机械叠激进静音会把话切碎（踩过「切话口切得太恐怖了」）。按逐字稿剪就 `silence_mode="off"`，只切口头禅，自然停顿全留。

2. **逻辑断了，先翻原片找料，别打补丁。** 预录视频删减后常不顺。第一反应不是加 overlay 硬补、也不是留着生硬——去看**原始素材完整逐字稿**，十有八九有句自然过渡被漏选了（grill-me 的「每个人想要的不同」、Swift 的「为什么没有用 Swift」都是这么捞回来的）。**真素材接 > overlay 补 > 留着断。** assemble 的 `inject` 就干这个。

3. **别信一次到位。** whisper 每次重剪都转出不同的错（grill me 转出 Google Me / GruMe / GroMe / GruelMe 四五个变体）。**每次重烧字幕都全量重扫**找新错，别以为上次纠过就干净了。

4. **渲完必验，wired ≠ working。** 写进 config ≠ 屏上对。每个 overlay 抽帧看、改过的字幕转录核。别拿「脚本返回 0」当「做对了」。

5. **overlay 锚语义、按遮挡定样式。** 位置锚在说那句话的时刻（anchor 关键词），不写死秒数；字幕只在真有结尾卡遮挡时才缩小。命令类「怎么开」、仓库名必须查实（创作者较真，见 command-howto.md）。

6. **创作者指哪改哪，其判断是 ground truth。** 创作者报一个秒数 / 一处别扭，就去那儿看清楚、精准改。用户显式取舍 > 默认参数。

## 两种落地形态

- **单条原片**（如 claude 变小龙虾）：一条录像，逐字稿≈原话，自动清理（删口水 / 然后 / 静音 / 死气口）。`silence_mode="aggressive"`。overlay 惯用老品牌卡模式（切封面卡做面板 + 命令药丸 + 结尾截图卡）。

- **逐字稿跨片拼接**（如小组件）：多条源片按批注过的逐字稿拼。`silence_mode="off"`。先跑 assemble stage 把源片按终稿拼成 assembled.mp4，`inject` 还原漏选的过渡句。overlay 惯用 logos / texts 模式。

> 调好的默认参数见 [defaults.md](defaults.md)（小龙虾磨出来的，别乱动）。
> 各 stage 的输入输出和 API 见 [pipeline.md](pipeline.md)。

## 教训：人称替换与验收要看「纠错后」文本（2026-07-11）
现象：留学篇(人物题材)成片里教授/同学全被写成「它」——流水线为 AI 视频焊死的全局「他→它」在人物题材上整体翻车;抽帧验收恰好没抽到带人称的句子,全文扫描扫的又是原始转录(替换发生在烧字幕时),两道检查都漏了。
判准：①片子主语是人不是 AI → config 必设 `"ta":"human"`;②验收字幕不能只扫原始转录,要用 build_fix(cfg) 把纠错链跑一遍再扫输出,或抽帧时专挑含人称/易错词的句子。
动作：开剪前问一句「这条讲人还是讲 AI」;验收清单加一项「过一遍纠错后全文」。

## 教训：反馈不是工单，成片前必须有批过的修改清单（2026-07-12）
现象：成品阶段创作者每报一个浅层问题(字幕跳动/慢读别扭),我就修那一个点+全片重渲一版,创作者被迫每轮重看7分钟找下一个问题;创作者最终点破「我每次只说了浅层问题,其实问题很多」。
判准：创作者的单条反馈=冰山一角的信号,不是完整需求;凡是要动全片渲染的改动,先问自己「修改清单批过吗」。
动作：收到反馈→先做全片系统诊断(节奏/内容/画面/音频/字幕逐维度过,附时间点)→合并创作者的反馈出完整修改计划(编号,像留学篇C1-C11)→创作者批注→一次执行一次渲染。渲染是最贵的一步,永远最后做。

## 教训：字幕分页必须按「自然断句」，不是按字数/间隔机械切（创作者强调过无数次，2026-07-11）
现象：动效层 VidCaption 的 `pages()` 用「间隙>0.6s 或积满 N 字」换页，结果**切断短语**（「第一如果可以的话问」）、还**把两句并一页**（把「解决它们」的尾巴和「今天这个是学业篇」并成「它们今天这个是学业篇」）。创作者原话：「你又没有根据句子的断句来，这个我说了好多次了」。
判准：字幕分页的**主边界 = whisper 分段（cap_segs）**——每段是一个自然停顿/短语，一段一页；只有单段过长（>~18 字）才在段内按标点再切，**绝不跨段合并**。
动作：`remotion_props.py` 给每个词打 `br`（段边界或超长处为 True），`VidCaption.pages()` 只按 `br` 分页，删掉字数/间隔启发式；烧死版 captions.py 同理先吃 cap_segs 边界。**每次重烧都抽帧核字幕分页，不只核文字对错。**

## 教训：字幕文字必须核对真实音频，绝不拿旧稿脑补（2026-07-11）
现象：创作者重剪后我重转写，whisper 把「全英教学」听成「已经教学」；我没核实这版实际说了什么，擅自照**旧 srt** 补成「上了国际学校全英教学」——编了这版没说的字。创作者当场抓：「高中三年都是全英教学，这个字幕错了」。
判准：corrections 只用来把 whisper 明显听岔的字改回创作者**真说过**的字；凡「加字/扩写」类改动（不是改错别字），拿不准这版说没说就**别按旧稿补全**，宁可保守、或听那几秒音频/问创作者。旧 srt 是别的剪辑版本，不是这版真值。
动作：补 corrections 时给「加字/扩写」类标存疑，渲染前听音频或问创作者核这几处。

## 教训:无声演示不止配速,还要过「内容体面」黑名单(2026-07-12)
现象:智能配速试剪把节奏做对了,但画面里的「Sign in again」登录横幅原样留着——算法只看变化率,没有内容语义。创作者一眼抓出。
判准:录屏素材进片前,OCR(--boxes 带位置框)扫黑名单词:Sign in/登录/密码/password/通知/update。命中=不宜展示。
动作:持续存在的横幅→按 OCR 坐标 drawbox 遮罩成背景色;瞬时弹窗→剪掉该帧段;拿不准的进修改计划给创作者看。配速(demo_pacing)与体面(blacklist)是两道独立工序,都过了才算演示段处理完。

## 教训:剪辑坐标必须文本查锁,不许凭记忆手写秒数(2026-07-12)
现象:开头试剪的落点段手写了旧时间轴的秒数(63.9,是重构前「发给它」的位置),成片最后3秒接错句;还把一句话的尾音切掉(手写21.4,文本查锁=22.4)。
判准:任何 pieces/editorial_cuts 的秒数,来源必须是「当前 transcript.lock 的文本查询结果」;凭印象写数字=必错(同一素材有多版时间轴)。
动作:剪辑脚本先 seg_of('锚句') 拿坐标再填 pieces;交付前用 props 验证「插段区窜词=0、尾段文本=预期句」。

## 教训:选材和删废话是剪辑的本职,不是越权(2026-07-13,创作者四连击)
现象:「如何找到并安装skill」把四条素材全用了,Playwright 被介绍两次(0497讲完0494又来)没发现;明显的废话一句没删;拿不准也没问——创作者点破:素材不代表都要用,要选好的用。
判准(内容三问,捋素材时每段必答):①这件事前面讲过吗(跑 redundancy_check.py,>0.55 报告全数进计划)②这句删掉观众损失什么(答不出=废话候选)③同一内容有几个 take 时,哪个最好(其余全弃,不做拼盘)。
动作:「不改创作者的原话」是字级红线;「删创作者的句子」是剪辑本职——敢删,删单进修改计划给创作者批。拿不准的删减=列成选项问创作者,绝不闷头保留。创作者说「剪映复查」不等于跳过诊断计划门禁——门禁管的是我的内容决策,复查管的是创作者的表现层加工,两回事。

## 静默区豁免了机器,豁免不了观众(2026-07-13)

60秒版结尾 11 秒静音(演示+尾卡),selfcheck 因为 quiet-zones 白名单没报警,创作者一看就说播放效果差。
教训:给自检加白名单时,想清楚豁免的是「误报」还是「真问题」——静默插段的冻帧检测是误报,
但死寂本身是真问题。**静默插段必须有 BGM/音效铺底**;selfcheck 已加「静默区声底」检查(⑤),
mean_volume < -50dB 报警,别再白名单掉。

## 观感层≠正确层(2026-07-13)

流水线的检查长期只有「对不对」(转录/映射/黑帧/响度/表意),没有「好不好看」:跳剪掩护、
声音设计、卡片动效、演示可读性、语速节奏。结构对了播放效果照样可以很差。出片前过一遍:
BGM 有没有/卡片动没动/演示看不看得清/接缝跳不跳/节奏拖不拖。

## 局部合法≠全局成立(2026-07-13 二审,四失误一病根)

演示留着(局部:有证据功能)、接缝放行(局部:连得上)、GitHub卡放行(局部:隔壁卡提过)、
1.07x(局部:无损伤)——每个局部检查都过了,观众全局体验全翻车。出片前必须跑
audience_read.py 出观众流水稿做全局审。另三条子规律:
- 从长版继承部件,按新时长预算重审性价比;
- 执行中发现违规,不许自我合理化——发现即上报创作者;
- 参数按目标观感定(给可感选项),不按「不被发现」定。
