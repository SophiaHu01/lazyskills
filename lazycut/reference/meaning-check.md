# 表意全文检查(剪视频最重要的是表意——创作者 2026-07-13 点破,借鉴 video-use 的 LLM Reasons 前置)

**时机(两次,都是强制):** ①选材捋顺后、逐字稿给创作者批注前——报告随逐字稿一起给创作者;②成稿剪完后、渲染前——把成稿字幕当一篇文章从头读到尾。

**怎么做(这是判断活,由主 agent 亲自做,不下放):**

1. **一句话主旨**:先写「这篇在说什么」。写不出一句=片子本身没立住,回结构层。
2. **逐段功能标注**:每段标 钩子/承诺/论点/论据/操作/例证/过渡/兑现/收尾;标不出功能的段=删除候选。
3. **逻辑链**:相邻段之间,前段的输出是不是后段的输入?哪里原地打转、哪里跳跃(跳跃处=需要桥:字幕卡/创作者补录半句/重排)。
4. **语义级重复**:redundancy_check.py 的机器报告**逐条表态**(采纳删哪个/为何保留)——机器报了不理=白装工具;再人工扫「不同措辞讲同一件事」。
5. **指代落地**:「这个/它/后面/那样」每处指代,小白观众能对上号吗。
6. **承诺兑现**:开头承诺的、中途预告的,片里给了吗(演示预告→演示在;「分几种情况」→数量对)。
7. **完备性**:缺收尾/缺证据/缺过渡,列成「留白清单」。
8. **废话密度**:每句问「删了观众损失什么」,答不出进删单。

**产出**:表意报告(主旨/功能表/问题编号)并入修改计划给创作者批。报告没批,不准渲染。

# 升级:剪前推理三层(创作者 2026-07-13 定的完整标准——表意只是及格线)

**第一层 表意**(上文八项)。
**第二层 爆款对标(强制,调 /video-flow)**:对最终稿跑 video-flow 的 11 条 checklist+对标模式+创作者账号实测数据——3秒钩/承诺句/中段起伏/高潮位/开环/首尾呼应/CTA/关注理由 逐条过;平台切分(抖音前20秒自成立?拆条候选?);选题层(教程题必须包装救,创作者的数据已验证)。**video-flow 是结构层武器,建了不跑等于没建。**
**第三层 最优成片推理(最深,主 agent 亲自做)**:选材完成后先别拼——问「这堆素材理论上能做出的最好视频是什么」:
  - 至少 2 个候选结构(不同开场/顺序/详略),各自对着第二层的 checklist 和创作者的数据打分;
  - 开场专项:素材里最强的 3 秒是哪句(结果前置/反差/共鸣,按创作者账号实测:教程式开场5秒流失74%);
  - 明确「这条素材做不到的」(缺收尾/缺证据),给补救选项而不是默默将就;
  - 结论=选定结构+理由+放弃的候选为何放弃,写进推理报告。
**产出顺序**:素材盘点 → 第三层(定结构) → 按结构拼稿 → 第一层(表意) → 第二层(对标) → 报告+逐字稿给创作者批 → 剪 → 成稿再过一二层 → 渲染。

# 剪后接缝检查(强制,2026-07-13 创作者点破「每秒信息量有了但逻辑断层」后立法)

信息密度刀有副作用:它按「这句自己有没有信息量」判死刑,看不见句子的**关节功能**。
「第一步就是」「那你把这个 prompt 丢出去之后」信息量≈0,但承担转场;删掉关节,前后脱臼。

**每一刀之后,对每条新接缝跑三步:**
1. **探测器**:后句以「也/还/另外/所以/那/这样子的话」开头,或含「这个/它」找不到先行词 → 判断层。
2. **朗读测试**:前句+后句连读,问「后句第一个词在新语境成立吗;前句语调收束了吗」。
3. **补桥,按优先级**:a. 挪剪点/从素材里找回创作者自己的原句当桥(最优,零违和);b. 章节小卡当路标掩护跳跃;c. 保留低密度关节句。

产出要求:接缝清单逐条给结论(✓/微瑕/断层),微瑕以上如实报创作者。

## 接缝探测器补丁(2026-07-13 二审)

- **条件分支着落**:条件句(「如果没有X」「如果说你需要X」)必然打开对偶分支(「有X呢?」)。
  删掉分支语音后,分支必须有着落——文字卡交代或语音找回,否则观众悬在半空。
- **局部合法≠全局成立**(本次四个失误的共同病根):缝连得上、卡有邻居、速度无损伤,都只是局部
  合法;必须再过一遍「观众流」全局审——见下。

## 成片多模态过审(强制,渲染后交付前)

观众接收的不是文字稿,是「语音+卡片+大字强调」的合流。用 audience_read.py 把 cap_segs 与
卡片 props 按时间轴合成「观众流水稿」,对它重跑表意八项——重点:指代/承诺/完备性/新名词是否
有头(卡片引入的词也算)。首犯案例:GitHub 语音被删后,类比卡解释了一个观众没听过的词。
