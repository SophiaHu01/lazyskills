# 卡片 vs 字幕强调:判断规律(2026-07-13 创作者给判例后立法)

创作者的原始判例:给观众讲 prompt → 显然要 text overlay 卡;说「直接让 AI 去安装」→ 不要卡,
字幕大字强调。推广成法:

## 上卡的条件(全满足才上)

信息是**观众要拿走/对照的参考物**:prompt 模板、步骤清单、类比解释(GitHub≈小红书)、
数据证据、无语音区的代行解说(演示段)。特征:超过语流一句话的容量、观众可能截图、
需要和语音并行阅读。

## 字幕大字标黄的条件

信息是**语流内的动作指令或重点词**:「请你帮我把这个东西进行安装」「免费」「99%」。
它是创作者正在说的话,不需要脱离语音独立存在。实现走 config `emphasis` 列表(现成机制)。

## 反例检验法(一票否决)

**卡上的字 ≈ 创作者正说的原话 → 这张卡违法,降级为字幕强调。**
(首犯案例:「找到后一句话:请帮我装上」卡,就是创作者原话复读,被创作者点名。)

## 密度红线

同屏至多一张卡;卡驻留 ≈ 阅读时长×1.5;60 秒片卡数 ≤5。
卡片一律走 Remotion 动效层(创作者批过的弹簧进出,VidCaption CardLayer),别用静态 PNG 硬切。

> 改动:2026-07-13 · Fable 5 · 初版(60秒版返工立的法)

## 卡片先行词规则(2026-07-13 二审补)

解释类卡(类比/术语解释)只能解释**语音或此前卡片已引入的词**;有资格引入新词的只有
框架卡/模板卡这类自我完备的参考物。执行中发现违规,**不许自我合理化放行——上报创作者拍板**。

## 短版结尾规约

≤60 秒的短版:结尾 = 步骤总结卡(拿走型)+尾卡 CTA;屏幕演示/长证据归长版或拆条,
不挤进短版(单位时长信息价值算不过账)。

## 语速参数

对标平台节奏(短视频 1.1-1.25x 常态),给创作者可感的选项(1.1/1.15),别按「不被察觉」定隐形参数。
