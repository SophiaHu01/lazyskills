# 动效规划(Motion Graphics 立法:先想,再渲)

> 目标对齐 ChatCut 的「一句话生成动画」:章节卡/快速图表/时间线/强调效果,不用 AE 不打关键帧。
> 我们的路线:小而一致的组件库 × 强规划纪律——每个动画都值得被想过,而不是有个卡片就贴。

## 组件库(motion/ 子项目,七种,禁止片内混用风格)

| 组件 | 用在什么时刻 | 关键 props |
|---|---|---|
| ChapterCard 章节卡 | 内容换章(「第二类…」「开始复刻」) | 编号/标题/副题 |
| BigNumber 数字揭示 | 口播报出关键数字(价格/百分比/倍数) | 数字/单位/一行说明,滚动计数进场 |
| QuickChart 快速图表 | 两个以上数字要对比(价格账/前后对比) | bar/line,{label,value}[],高亮项 |
| StepsTimeline 步骤时间线 | 讲流程/步骤(三步法/流水线) | steps[],当前步高亮,逐步点亮 |
| Callout 指点强调 | 「看这里」——指向画面某区域 | 类型(箭头/圈/下划线)/归一化坐标/短标签 |
| ListReveal 列表逐条 | 并列要点逐条弹出(已有,原模板卡升级) | title/items[],随口播节拍逐条 |
| QuoteCard 引用卡 | 引用原话/金句定格 | 文本/出处 |

## 规划工序(强制,在逐字稿锁定之后、渲染之前)

1. **通读逐字稿,标「动画时刻」**。判断标准(按优先级):
   - 口播出现**数字** → BigNumber;**两个以上数字** → QuickChart(数据必须来自口播或已核实事实,不许编);
   - 口播讲**步骤/流程** → StepsTimeline;
   - 口播说**「看这里/这个位置」**或录屏里有观众该看的点 → Callout(坐标从视觉台账查,不手写);
   - **换章** → ChapterCard;并列要点 → ListReveal;金句/引用 → QuoteCard。
2. **产出动效脚本(motion_plan.json)**:每条={组件, 内容/数据, 锚句(定时用,不写秒数), 时长, 为什么}。
   「为什么」必须写——写不出理由的动画删掉。
3. **密度红线**:同屏至多一个动效;相邻动效间隔≥3s;60 秒片动效总数≤8;
   连续两个同类型要有理由。动画服务理解,不是装饰——疑罪从删。
4. **动效脚本随修改计划给创作者批**,批完才渲。
5. 渲染:motion_plan.json → props → Remotion 透明层(四 flag) → 合成;组件样式 token 全部
   读 assets/style.json(观众换风格=动效跟着换)。

## 一句话生成(协作车道)

创作者说「这里来一个价格对比动画」→ AI 选组件、从上下文取数、填 props、渲染——
零关键帧。数据取不到就问,绝不编造(图表里的每个数字都要有出处)。
