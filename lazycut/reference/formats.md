# 格式备案:输入/内部/输出/平台 四层标准(单一真相源)

> 所有格式相关的判断查这里,别散落记忆。改标准=改这份文件。

## 一、输入素材(什么能进厂)

| 项 | 标准 |
|---|---|
| 容器 | mp4 / mov / MOV(其他容器 ffmpeg 能解就能进,首选这三) |
| 口播素材 | 竖屏横屏均可;多条拼接时统一归一到首条的画幅 |
| 归一化 | 拼接时统一 30fps / 48kHz 立体声 / setsar=1;竖屏默认 1080x1920 |
| 录屏素材 | 任意分辨率(处理时按活动区裁切);OCR 台账按 2s 间隔采样 |
| 音频 | 视频内嵌音轨;外部麦轨用文本对齐法对位 |

## 二、内部时间线(厂内通用语言)

| 项 | 标准 |
|---|---|
| 时间单位 | 秒(浮点,精确到 0.001);CapCut 草稿层换算为微秒 |
| 帧率基准 | 30fps(EDL 时码按此换算) |
| 坐标来源 | 一律 transcript.lock 文本查询,禁止手写秒数 |
| 变速表达 | cutplan.seg_speeds / 草稿 segment.speed;素材永不预剪碎 |

## 三、输出出口(三类编辑器地图)

| 类 | 代表软件 | 通道 | 状态 |
|---|---|---|---|
| 明文可写 | 剪映/CapCut(桌面版) | 原生草稿(模板克隆法,微秒,身份链自检) | ✅ 最深,mac 实测/Win 征集 |
| 封闭但有官方大门 | DaVinci Resolve / Premiere / Final Cut | EDL(CMX3600,30fps,M2 变速备注)/ OTIO(需 opentimelineio;FCPXML 经其 adapter) | ⚠️ beta,按规范生成,各 NLE 真机导入征集中 |
| 真封闭无通道 | iMovie / 手机剪辑 app | 分层素材包:干净底片 mp4 + SRT(UTF-8)+ 卡片 PNG(RGBA)+时间表 | ✅ 万能兜底 |
| 直接成品 | — | mp4(H.264 CRF19 / AAC 192k) | ✅ |

**剪映特别注**:它是唯一无官方导入通道的主流软件(不认 EDL/XML),故必须写原生草稿;
国际版主文件 draft_info.json、国内版 draft_content.json,自动识别,格式以用户本机模板为准。

## 四、平台交付标准(成片必须满足)

| 项 | 标准 | 依据 |
|---|---|---|
| 响度 | 整体 -14 LUFS(±1.5 报警),TP -1.5 | 各平台通用标准,selfcheck ④ 把关 |
| 竖屏画布 | 1080x1920 | 抖音/小红书/Shorts 通用 |
| 安全区 | 左右各 ≥130px(右侧点赞栏);字幕底部偏移见 style.json | 平台 UI 遮挡实测 |
| 字幕断行 | 竖屏 ≤13 字/行,横屏 ≤22(style.json 可覆盖);不断英文词 | 可读性实测 |
| 静默区 | 必须有声底(mean_volume ≥ -50dB),selfcheck ⑤ 把关 | 死寂=观众以为坏了 |
| 预览压缩 | 720p / CRF28(IM 传输 <29MB 为宜) | 传输限制 |
